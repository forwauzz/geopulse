GEO-Pulse — UX/UI Constitution v1

Owner: Product Architect
Status: Active
Scope: Governs all frontend UX/UI decisions
Applies to: Landing page, audit flow, results UI, dashboards, and future features

1. Product Intent

GEO-Pulse is a precision AI visibility diagnostic tool.

It exists to:

Reveal how AI systems interpret and rank a website
Surface structural visibility gaps
Provide clear, prioritized fixes
Deliver insight in under 60 seconds

GEO-Pulse does NOT exist to:

Feel like a generic SEO tool
Showcase AI for its own sake
Entertain or gamify diagnostics
Overwhelm users with dashboards or metrics
2. Core Optimization Target

GEO-Pulse optimizes for:

Clarity of insight with minimal cognitive effort

Every UX decision must reduce:

Time to understanding
Interpretation effort
Decision friction
Noise during analysis

If a screen requires “figuring out,” it fails.

3. Product Positioning (UX Translation)

GEO-Pulse is not:

A dashboard
A marketing tool
A content platform

It is:

A diagnostic instrument

The UI must feel:

Precise
Structured
Fast
Credible
4. Brand & Visual System
4.1 Visual Tone

The interface must feel:

Technical, not trendy
Calm, not loud
Intelligent, not flashy
4.2 Color System

Primary palette (recommended):

Background: Off-white / very light grey
Primary: Deep navy (#1E2A38)
Accent: Cyan (used sparingly for interaction)
Text: Charcoal / near-black
Color Rules

Accent color is used ONLY for:

Primary actions
Focus states
Minimal highlights

Never for:

Decorative gradients
Background fills
Large visual areas
4.3 Visual Restrictions

The UI must NOT include:

Gradients
Neon colors
Overuse of icons
Decorative illustrations
“Startup-style” fluff
5. Typography Doctrine

Two-layer system:

Layer 1 — System Layer (Sans Serif)

Used for:

Navigation
Labels
Tables
Metadata
UI controls

Purpose:

Clarity and speed
Layer 2 — Emphasis Layer (Optional Serif or Stylized Sans)

Used for:

Hero headlines
Key statements

Purpose:

Authority and focus

Typography must never:

Compromise readability
Feel playful or casual
6. Structural Doctrine
6.1 Structure Before Feature

Every screen must clearly communicate:

Primary action
Current state
Output or result

If unclear within 3 seconds → redesign.

6.2 Layout Discipline

All layouts must be:

Grid-aligned
Balanced
Predictable

Avoid:

Floating elements
Asymmetry without purpose
Layout inconsistency between pages
6.3 Page Types (Canonical)

GEO-Pulse has 3 primary surfaces:

1. Landing (Input)

→ Capture URL
→ Frame value

2. Processing (Transient)

→ Minimal feedback
→ No loading theatrics

3. Results (Core Product)

→ Score
→ Issues
→ Fixes

7. Interaction Laws
7.1 Calm Authority Language

All text must be:

Direct
Clear
Confident

Examples:

✔ “Run diagnostic”
✔ “Analysis complete”
✔ “3 critical issues detected”

Avoid:

✘ “Let’s analyze your site!”
✘ “AI is thinking…”
✘ “Processing magic…”

7.2 No AI Theater

Never:

Over-label AI usage
Show confidence scores unnecessarily
Explain the model unless required

Users should feel:

“This tool is smart”

Not:

“This tool is AI”

7.3 Fast Feedback

The product must feel:

Immediate
Responsive

If delays occur:

Use minimal indicators
No progress bars unless meaningful
8. Input Experience Rules

The input is the entry point and must feel:

Simple
High-value
Intentional
Input Design Rules
Single field focus
Clear placeholder: “Enter website URL”
Strong CTA: “Run diagnostic”
Pre-Result Framing (Required)

Before submission, user must understand:

“You will receive:”

Visibility score
Key issues
Priority fixes
9. Results Experience (Core UX)
9.1 Results Must Answer Immediately

Within 3 seconds, user must see:

Score
Status (good / needs improvement / critical)
Number of issues
9.2 Hierarchy of Results
1. Score (top)
2. Key issues (middle)
3. Fix recommendations (bottom)
9.3 Issue Presentation

Each issue must include:

Title (clear problem)
Impact level
Short explanation
Actionable fix

No paragraphs. No fluff.

9.4 Fixes

Fixes must be:

Prioritized
Concrete
Immediately usable
10. Cognitive Load Rules

GEO-Pulse must reduce:

Scanning effort
Interpretation effort
Decision-making time

Never:

Show all data at once
Overload with metrics
Add unnecessary toggles
11. Trust & Credibility Layer

The product must feel:

Reliable
Deterministic
Professional

Achieved through:

Clean structure
Clear outputs
No hype language
12. Conversion Philosophy

Conversion is driven by:

Clarity of value
Speed of insight
Perceived authority

Not by:

Aggressive CTAs
Popups
Urgency tricks
13. Definition of Done (UX Rubric)

Every UI must pass:

Is the primary action obvious within 3 seconds?
Is the output immediately understandable?
Does this reduce cognitive effort?
Does it avoid AI theater?
Does it feel like a diagnostic tool (not SaaS)?
Is visual noise eliminated?

If any answer is “no” → not complete.

14. Future Expansion Rule

Any new feature must:

Have a single clear purpose
Improve clarity or speed
Not introduce cognitive weight

If it adds complexity without clarity → reject.

15. Final Principle

GEO-Pulse is not a UI.

It is an instrument for understanding AI visibility.

Every design decision must reinforce this.
